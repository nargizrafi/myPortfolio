Understanding CSS Methodology : BEM

CSS Methodologies

CSS is an abbreviation that stands for Cascading Style Sheets. Simply put, it enhances a website's visual appeal beyond that of plain or monotonous text. Visual structure, layout, and aesthetics are all set by CSS.

CSS is a simple language to learn and use, but in more complex projects, without a set norm, the code becomes messy. In order to solve this problem, different CSS methodologies have been created. CSS without established methodology can make scaling and maintaining your applications difficult in the long run. CSS methodologies offer a systematic and organised way to create, arrange, and maintain CSS code. Particularly in collaborative development environments and for projects with a substantial amount of stylistic requirements, they are crucial for building stable, scalable, and well-organized web interfaces. Besides these, a standardised methodology also makes it easier for everyone to comprehend how styles are arranged, identified, and used when several developers are working on a project. There are a number of widely used CSS methodologies, each with an individual goal, and one of them is BEM.

BEM (Block-Element-Modifier)

BEM stands for Block-Element-Modifier. The methodology was created by Yandex to improve the connection between HTML and CSS files and to make developers use more detailed class names. BEM has a lot of advantages, including improving code readability and reusability. Before we talk about it, we need to understand the structure of BEM.

There are three main parts of BEM:

Block : An independent component or higher-level container with a clearly defined UI function. A block can also contain additional blocks and elements. It follows the naming convention in CSS as:

                        
        .block {
            ... 
        }
                        
        /* Blocks are named as standard CSS classes */

Element : It is a part of the block that serves a particular purpose. Elements are always confined within blocks. It follows the naming convention in CSS as:

                        
        .block__element {
            ...
        }

        /* Elements are declared by two underscores after the block */

Modifier : Represents a variation or state of a block or element. Blocks or elements can have their behaviour or appearance changed with modifiers while maintaining their primary functionality. It changes the default presentation of a block or element. It follows the naming convention in CSS as:

                        
        .block--modifier {
            ...
        }
                        
        .block__element--modifier {
            ...
        }
                            
        /* Modifiers are declared by 2 dashes after the block or the element */

To summarize, for example, if we have a div component, it might look like this:

HTML

                        
        <div class="block block--modifier">

        <button class= "block__element block__element--modifier" > Click me </button>

        </div>

CSS

                        
        .block {...}
        .block__element {...}
        .block--modifier {...}
        .block__element--modifier {...}

Another example:

                        
        <nav class="nav">
            <ul class== "nav__list">
                <li class="nav__item"><a class="nav__link"  href="#">Home</a></li>
                <li class="nav__item"><a class="nav__link nav__link--active" href="#">About</a></li>
                <li class="nav__item"><a class="nav__link" href="#">Services</a></li>
            </ul>

In this example:

• nav is the block.
• nav__list, nav__item, and nav__link are elements.
• nav__link--active is a modifier indicating the active link.

Advantages and Disadvantages

As you can see, this makes the code easier to read, and makes it easier for developers to understand the connections between various components.

From small websites to large web applications, BEM works effectively for a range of projects. As your project expands, maintaining and upgrading your codebase will be easier to manage if you have a clear naming system and organised styles. As a result, by using BEM the code also becomes simpler to update and maintain.

On the other hand, there are some criticisms about BEM:

  • When working with deeply nested components and modifiers, class names can be very long and complex.
  • BEM can cause some complexity in terms of class names and markup, and it can make HTML and CSS files larger than expected.
  • It may be necessary to rework and modify current styles in order to implement BEM in an existing project, which can take time.

But we can say that, in the end, the decision to adopt BEM CSS depends on the requirements of the project, the team's expertise with the methodology, and the balance between its benefits and drawbacks in the particular situation. In my opinion, BEM is a highly effective methodology as well as other methodologies. Its advantages like modularity and reusability, still make BEM a good choice for projects of varying sizes.
